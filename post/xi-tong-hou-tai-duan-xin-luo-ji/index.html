系统后台短信逻辑

2019-11-26 Views 2019 实习 3 min read

一个后台短信逻辑包括：

  1. 第三方短信发送平台接入（阿里云...）
  2. 系统内的短信模版建立
  3. 系统内的短信模版审核
  4. 操作验证和登陆认证对短信模版的调用

应该如何设置接入

首先需要的是对应平台的Key和_的填写项，可以让用户自行选择合适平台配置，并加入测试按钮，以测试配置的信息是否正确，能否连接对应平台。正常连接后点击确定，随后在合适位置显示绿色「 已接入 」的字段提示用户。操作错误显示的是灰色的「 未接入 」字段

短信模版显示的内容

  1. 模版唯一标识（编码或者ID）
  2. 模版的标题名称（点击可以预览模版）
  3. 模版的内容信息
  4. 模版审核的状态（未通过审核时可以查看原因）
  5. 模版的状态（可以进行开关）
  6. 新增按钮（可以编辑名称-签名-类型-内容）其中内容会根据类型改变而改变，最常见的就是验证码和短信通知这两种类型；其中验证码只有name这个变量可以更改，而短信通知可以更改name-num-time这三个变量，name可以是用户名称，也可以是账号……；num主要是订单号-密码……；time是指会议时间……
  7. 编辑按钮（仅可以编辑模版标题-签名-内容信息）
  8. 新增和编辑都可以即时预览生成的模版和删除模版

模版审核显示的内容

  1. 审核类型（新增模版-编辑模版-删除模版）
  2. 模版标题名称（点击可以预览）
  3. 内容概览（显示不完全则……，完整点击名称即可查看）
  4. 审核提交时间
  5. 操作（通过-不通过）不通过则要填写不通过的原因

操作验证和登陆认证的内容

  1. 调用的接口编码-接口名称（会匹配是否为同一个接口，不正确则提醒用户修正）
  2. 使用的短信模版编码-短信模版名称（会匹配是否为同一个模版，不正确则提醒用户修正）
  3. 验证/认证的状态开关（关了就不需要通过验证就可以进行某些内容修改）
  4. 测试按钮（测试接口是否存在以及有没有配置错误）
  5. 预览按钮（预览以确认选择的模版是否正确）
