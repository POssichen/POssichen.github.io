Mac os Visual Design 视觉设计

2020-01-30 Views 2 min read

Animation 动画

  1. 追求显示主义。现实的运动可以帮助人们了解事物的运作方式。
  2. 避免将动画用于频繁发生的交互。重复出现的动画会导致用户认为动画降低了效率。
  3. 增强反馈和理解。例如最小化窗口时不是突然消失，而是从屏幕流畅地移动到Dock，以便用户知道它的去向。
  4. 中对象之间平滑过度。
  5. 暗示结果。例如将一个项目拖到Dock栏，其他项目会移动到一边，显示即将放置的位置。
  6. 确保全屏模式过渡动画流畅。
  7. 明智地使用动画和动作效果。
  8. 将动画设置为可选。

Color 颜色

  1. 明智地使用颜色进行交流。
  2. 在不同国家使用适当含义的颜色。
  3. 避免使用存在歧义的颜色。
  4. 考虑半透明颜色的使用场景。
  5. 在各种光照条件下测试应用的配色方案。
  6. 允许用户使用标准的颜色面板来更改颜色。
  7. 不要在应用程序中对系统颜色进行硬编码。
  8. 不要重新定义系统颜色的定义。

Dark Mode 黑暗模式

  1. 确保所有外观都具有足够的色彩对比度。
  2. 柔化白色背景的颜色。
  3. 尽可能使用模版影像。
  4. 使用系统提供的标签颜色来显示文本。
  5. 使用系统视图绘制静态文本。

Translucency 半透明

  1. 首选系统提供的元素和颜色。
  2. 使用系统模版图像。

Typography 版式

  1. 使用字体粗细、大小和颜色突出显示应用程序中最重要的内容。
  2. 确保自定义字体清晰易读。
  3. 使用正确的字体变体。
  4. 使用标准的字体面板更改用户字体。
  5. 使用动态系统字体变体来匹配标准控件的文本外观。
