Mac os User Interaction 用户互动

2020-02-07 Views 4 min read

Authentication 认证方式

  1. 尽可能延迟登陆。用户在不得不做任何有用的操作前被迫登陆通常会选择放弃应用程序。
  2. 说明身份验证的好处以及如何注册服务。

Data Entry 数据输入

无论是使用键盘、鼠标、触控板还是语音，输入信息都可能是一个乏味且容易出错的过程。

  1. 让人们尽可能地做出选择。考虑使用表格、弹出按钮或者单选按钮组替代文本字段。使数据输入的更快更高效。
  2. 简化列表导航。
  3. 使用介绍性标签或文本来达到帮助目的。
  4. 适当调整文本字段换行符。
  5. 考虑使用拓展工具提示来显示已剪切或截断的文本。
  6. 适当使用安全的文本字段。如输入密码时。

Prepopulating Data 预填充数据

  1. 尽可能从系统获取信息。尽量不要强迫人们提供可以自动收集或在用户许可下收集。
  2. 提供合理的默认值。
  3. 不要预先填充密码字段
  4. 确保应用填充数据的准确性

Validating Data 验证数据

  1. 仅在必要时才需要字段值。
  2. 推迟显示必填指示符。
  3. 动态验证字段值。尽可能在输入后立即检查字段值，以便用户立即进行改正

Drag and Drop 拖放

  1. 尽可能使拖放操作可逆。
  2. 确定拖放的结果是移动还是复制。
  3. 尽可能让人们一步选择并拖动内容。
  4. 用户可以在非活动窗口拖动选定内容。
  5. 拖动图像时在指针下方显示拖动图像。
  6. 在多项目拖动过程中显示项目数量。
  7. 明确指出何时无法删除。
  8. 将内容拖到新目标上方时，突出显示该目标。
  9. 插入拖放时显示拖放内容出现的位置。
  10. 如果拖放失败，则使用预期的动画。
  11. 完成无法撤销拖放操作前请确认。

File Handling 文件处理

用于用户与文件系统进行交互以打开和保存文件

  1. 确定打开的是工作表还是独立窗口。
  2. 默认保存为合理位置。如果启用了iCloud同步，那iCloud文件夹通常为最佳默认位置。
  3. 考虑添加描述性标题或消息。
  4. 让人们以预期的方式打开文档。
  5. 尽可能允许选择多个文件。
  6. 尽量包含合理的过滤器，以便用户选择所需要的文件。

Help 帮助

用户有时需要帮助来学习高级和辅助功能，可以通过帮助标签和Apple帮助的形式提供帮助。

  1. 避免限制用户操作。
  2. 尽描述指针正下方的控件。
  3. 将帮助标签添加到特定于应用程序的控件中
  4. 尽可能少的单词来描述控件。
  5. 考虑提供上下文相关的帮助标签。
  6. 提供基于任务的描述。

Keyboard 键盘

  1. 遵守标准的键盘快捷键。
  2. 为标准菜单项启用预期的键盘快捷键。
  3. 请勿覆盖标准键盘快捷键。
  4. 保留预期的键盘快捷键。

Mouse and Trackpad 鼠标和触控板

  1. 使用标准控件和视图。
  2. 根据上下文和含义对点击和手势做出响应。
  3. 避免重新定义系统范围内的应用间手势。
  4. 避免重新定义反馈的类型。

Providing User Feedback 提供用户反馈

  1. 将状态和其他类型的反馈集成到界面中。
  2. 可能导致数据意外丢失时警告他们。
  3. 告诉用户什么时候不能执行命令。

Requesting Permission 请求权限

  1. 仅在应用明确需要时才请求个人数据。
  2. 请求时需说明为何需要这些信息。
  3. 仅在应用运行时才请求权限。
  4. 不要请求不必要的位置信息。
